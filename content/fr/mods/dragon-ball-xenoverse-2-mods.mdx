<Section icon={Shield} title="Rétrograder votre version du jeu via la console Steam">

Lorsqu'une nouvelle mise à jour sort, Steam met automatiquement à jour vos fichiers locaux. Pour restaurer la compatibilité de vos **mods Dragon Ball Xenoverse 2**, vous devez télécharger manuellement un dépôt plus ancien. Ce processus utilise la console de ligne de commande intégrée de Steam pour demander des manifestes plus anciens directement depuis les serveurs officiels.

| Composant | Identifiant | Objectif |
|-|-|-|
| App ID | 454650 | Identifie le jeu principal sur la plateforme Steam |
| Depot ID | 454651 | Identifie le package de contenu spécifique à télécharger |
| Manifest ID | Variable | Identifie le correctif de version historique spécifique |

<Steps>
  <Step num="1" title="Ouvrir la console Steam">
    Appuyez sur les touches Windows + R de votre clavier pour ouvrir la boîte de dialogue Exécuter. Tapez `steam://open/console` et appuyez sur Entrée. Cette action ouvre un onglet de console dédié au sein de votre client Steam actif.
  </Step>
  <Step num="2" title="Localiser l'ID du manifeste">
    Naviguez sur SteamDB dans votre navigateur Web. Recherchez l'App ID 454650, sélectionnez l'onglet Depots (Dépôts) et choisissez le dépôt de contenu principal. Parcourez l'onglet d'historique (History) pour trouver l'ID de manifeste associé à votre version cible, telle que la version 1.25.02.
  </Step>
  <Step num="3" title="Exécuter la commande de téléchargement">
    Dans la console Steam, saisissez la commande de téléchargement en utilisant la syntaxe suivante : `download_depot 454650 454651 [ID_Manifeste]`. Appuyez sur Entrée pour commencer le téléchargement en arrière-plan.
  </Step>
  <Step num="4" title="Remplacer les fichiers du jeu">
    Une fois que la console affiche un message confirmant que le téléchargement est terminé, accédez au chemin du dossier désigné. Copiez les fichiers téléchargés et remplacez les fichiers de votre répertoire principal.
  </Step>
</Steps>

<Callout type="info" title="Emplacement de téléchargement Steam">
Steam télécharge les dépôts de la console dans votre répertoire d'installation par défaut sous `Steam\steamapps\content\app_454650\depot_454651`. Assurez-vous de copier le contenu de ce dossier, et non le dossier lui-même, dans votre répertoire principal.
</Callout>

</Section>

<Section icon={Swords} title="Installation des outils de modding essentiels">

L'écosystème du modding repose sur deux composants principaux : l'Eternity Patcher, qui contourne les vérifications de fichiers, et le Mod Installer, qui gère l'intégration des fichiers.

Pour vous assurer que ces utilitaires fonctionnent correctement, passez en revue les différences de fonctionnement et les exigences spécifiques des deux principaux outils de modding :

| Outil utilitaire | Format de fichier principal | Dépendance clé | Répertoire d'installation |
|-|-|-|-|
| Eternity Patcher | .ini / .dll | Répertoire XV2Patcher | Dossier racine du jeu |
| XV2INS | .x2m | xv2ins.exe | N'importe où (Lien vers bin/xinput1_3.dll) |

Pour installer ces outils, extrayez l'archive de l'Eternity Patcher directement dans votre répertoire racine. Le dossier racine est l'endroit où se trouve l'exécutable de votre jeu. Ensuite, lancez l'application XV2INS et pointez-la vers l'exécutable de votre jeu lorsque vous y êtes invité. Cela établit la connexion requise pour écrire de nouveaux fichiers dans le répertoire de votre jeu.

<Callout type="tip" title="Sauvegardez d'abord vos fichiers de sauvegarde">
Avant de lancer l'installateur pour la première fois, faites une copie de votre fichier de sauvegarde situé dans le répertoire Steam Userdata. Les modifications de configuration des mods peuvent parfois corrompre les données de sauvegarde.
</Callout>

<Section icon={Map} title="Gestion et lancement du contenu personnalisé">

Une fois le framework installé, vous pouvez commencer à ajouter du contenu personnalisé. Les mods pour le jeu se présentent généralement au format `.x2m`, qui contient tous les modèles, textures, animations et données statistiques nécessaires. Gérer correctement ces fichiers évite les conflits qui mènent à l'instabilité du jeu.

| Catégorie de mod | Impact sur la stabilité | Ordre de chargement recommandé | Conflit de fichiers clé |
|-|-|-|-|
| Personnages personnalisés | Faible | N'importe quel ordre | Collisions d'ID d'emplacement |
| Mods de refonte (Overhaul) | Élevé | Installer en premier | Écrasement des fichiers système |
| Effets visuels | Moyen | Installer en dernier | Compatibilité des shaders |

Le modding peut parfois entraîner des erreurs dues à des problèmes de chemin de fichier ou à des écarts de version. La plupart des problèmes peuvent être résolus en vérifiant attentivement l'emplacement des fichiers et les paramètres de configuration.

| Description de l'erreur | Cause probable | Solution |
|-|-|-|
| Plantage au démarrage | Version du patcher incompatible | Rétrograder le jeu ou mettre à jour le patcher |
